# team-lead-router

Маршрутизатор: любая задача, связанная с программированием, передаётся агенту
[`team-lead`](../../agents/team-lead.md). Сам team-lead уже решает, какой
пайплайн запускать — backend, frontend, QA, security и т. д.

## Триггеры

Код, баг, фича, тесты, деплой, миграция, API, фронтенд, бэкенд, CI/CD, Docker,
pull request, debug, endpoint, компонент, модуль, npm, pip, git (в контексте
разработки).

## НЕ триггеры

Документы, презентации, PDF, общие вопросы без контекста разработки — для них
есть отдельные скилы (`docx`, `pptx`, `pdf`).

## Зачем нужна прослойка

Без team-lead задачи решаются в изоляции — без ревью, без тестов, без учёта
архитектуры. С ним каждая задача проходит через корректный процесс: план,
делегирование специализированным агентам, сбор результата.

## Зависимости

- Сабагент `team-lead` в `~/.claude/agents/team-lead.md` (плюс набор
  специализированных агентов, на которых он опирается).
