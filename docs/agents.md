# Дизайн сабагентов

## Когда нужен скил, а когда агент

- **Скил** — это «тригер + инструкция». Активируется автоматически по
  description, исполняется в основной сессии Claude.
- **Агент** — это самостоятельная подсессия с собственным контекстом и
  набором инструментов. Запускается через `Agent` tool.

Правило: если задача требует своего контекста, многошагового процесса или
изоляции — это агент. Если хватает короткой инструкции на основной сессии —
это скил.

## Паттерн «router → agent»

Удобный паттерн: лёгкий скил с агрессивным `description` ловит запрос и
немедленно делегирует одноимённому агенту. Так срабатывание гарантировано, а
вся сложная логика живёт в агенте, где её удобнее ограничить инструментами и
моделью.

Примеры в репо: `commit` (скил) → `commit` (агент), `team-lead-router` →
`team-lead`.

## Когда заводить нового агента

- Появляется задача, которую делает несколько разных скилов одинаково —
  выноси в агент.
- Нужен отдельный контекст / модель / набор инструментов.
- Пайплайн состоит из 3+ шагов с собственной памятью.

## Чего избегать

- Один агент = один ответственный участок. Не делай «универсального бога».
- Не давай агенту больше инструментов, чем ему нужно (`tools:` в frontmatter).
- Не вызывай агентов рекурсивно без чёткого условия выхода.
